# 修仙游戏功能说明文档

## 1. 游戏概述

这是一款基于HTML5的文字放置类修仙游戏，玩家通过不断修炼、提升境界、收集装备和灵宠来增强实力，最终成为修仙界的顶尖存在。游戏采用模块化设计，包含多个独立系统，支持本地存储游戏进度。

## 2. 核心系统

### 2.1 角色系统

**功能说明**：
- 角色创建：玩家可以输入道号并选择出身身份，不同身份有不同的属性加成
- 属性管理：角色拥有根骨、悟性、机缘、身法四个基础属性，可通过消耗自由属性点进行调整
- 身份选择：提供多种出身身份，每种身份有独特的属性加成

**操作流程**：
1. 首次进入游戏时创建角色
2. 选择身份（如凡人、富家子弟、宗门弟子等）
3. 输入道号
4. 点击「开始修仙」按钮完成创建

**属性加成**：
- 根骨：影响修炼效率
- 悟性：影响功法参悟速度
- 机缘：影响掉落和暴击率
- 身法：影响战斗速度和闪避率

### 2.2 修炼系统

**功能说明**：
- 修为修炼：自动获得修为，提升境界
- 炼体系统：提升肉体强度，增加属性
- 修炼加速：使用灵石或观看广告加速修炼

**操作流程**：
1. 点击「开始修炼」按钮开始修炼
2. 修炼过程中可以点击「修炼效率」查看当前修炼速度
3. 点击「加速」按钮使用加速功能
4. 点击「停止修炼」按钮停止修炼

**修炼效率公式**：
- 基础修炼效率 = baseExpGain
- 当前修炼效率 = 基础修炼效率 * (跟脚加成 + 功法加成)

**加速选项**：
- 1小时加速：消耗100灵石
- 8小时加速：消耗500灵石
- 24小时加速：消耗1000灵石
- 永久加速：VIP特权

**炼体系统**：
- 炼体间隔：30秒
- 炼体效率：根据角色属性计算
- 炼体收益：获得炼体经验，提升肉体强度

### 2.3 境界系统

**功能说明**：
- 境界突破：当修为达到要求时可以突破到更高境界
- 境界加成：不同境界提供不同的属性加成
- 层数提升：每个境界包含多个层数，提升层数需要累积修为

**境界列表**：

| 境界 | 描述 |
|------|------|
| 炼气期 | 修仙入门 |
| 筑基期 | 基础稳固 |
| 金丹期 | 凝结金丹 |
| 元婴期 | 元婴成形 |
| 化神期 | 元神出窍 |
| 合体期 | 身心合一 |
| 渡劫期 | 渡过大劫 |
| 大乘期 | 大乘境界 |
| 真仙 | 飞升仙界 |
| 金仙 | 金仙境界 |
| 仙帝 | 仙界巅峰 |

**突破流程**：
1. 当修为达到当前境界上限时，突破按钮会变为可点击状态
2. 点击「突破」按钮尝试突破
3. 突破有一定成功率，失败会积累突破经验
4. 突破成功后进入下一境界

### 2.4 装备系统

**功能说明**：
- 装备获取：通过探索、任务等方式获得装备
- 装备强化：消耗强化石和灵石提升装备等级
- 装备洗炼：消耗洗练石重置装备属性
- 装备分解：分解不需要的装备获得强化石

**装备部位**：
- 武器：增加攻击力
- 衣服：增加防御力
- 腰带：增加生命值
- 鞋子：增加速度
- 项链：增加特殊属性
- 戒指：增加特殊属性
- 玉佩：增加特殊属性
- 法宝：增加特殊属性

**装备品质**：

| 品质 | 颜色 | 描述 |
|------|------|------|
| 凡品 | 灰色 | 基础属性 |
| 良品 | 绿色 | 略高属性 |
| 精品 | 蓝色 | 较高属性 |
| 绝品 | 紫色 | 高属性 |
| 神器 | 橙色 | 顶级属性 |

**强化系统**：
- 强化等级上限：15级
- 强化成功率：随等级提升而降低

| 等级范围 | 成功率变化 |
|----------|------------|
| 1级 | 100% |
| 2-5级 | 每级降低5% |
| 6-11级 | 每级降低10% |
| 12-15级 | 每级降低2% |

- 强化消耗：随等级提升而增加
  - 强化石：5 * 1.5^level
  - 灵石：100 * 1.1^level

**分解系统**：
- 分解装备获得强化石
- 返还数量：基础返还 + 强化等级 * 2

### 2.5 灵宠系统

**功能说明**：
- 灵宠获取：通过任务、活动等方式获得灵宠
- 灵宠培养：消耗灵石提升灵宠等级和属性
- 灵宠出战：选择一只灵宠出战，获得属性加成

**初始灵宠**：
- 小火龙（普通）：基础攻击型灵宠
- 小凤凰（稀有）：平衡型灵宠
- 小麒麟（史诗）：高级全能型灵宠

**灵宠属性**：
- 攻击：增加玩家攻击力
- 防御：增加玩家防御力
- 生命值：增加玩家生命值
- 速度：增加玩家速度

**培养流程**：
1. 选择已获得的灵宠
2. 点击「培养」按钮
3. 消耗灵石提升灵宠等级
4. 灵宠等级提升后，属性会随机增加

**培养消耗**：
- 灵石：灵宠等级 * 100

### 2.6 丹炉系统

**功能说明**：
- 丹药炼制：消耗材料炼制丹药
- 丹炉升级：提升丹炉等级，增加炼丹成功率
- 丹药使用：使用丹药获得各种效果

**丹药类型**：
- 疗伤丹：恢复生命值
- 灵气丹：增加灵气
- 修为丹：增加修为
- 洗髓丹：重置属性点

**炼制流程**：
1. 选择要炼制的丹药配方
2. 输入炼制数量
3. 检查材料是否足够
4. 点击「开始炼制」按钮
5. 等待炼制完成
6. 获得丹药（数量受成功率影响）

**成功率计算**：
- 基础成功率：10%
- 丹炉加成：根据丹炉等级增加
- 功法加成：根据已学习的功法增加
- 总成功率上限：95%

**炼制时间**：
- 基础时间：2秒/颗（一阶）
- 每升一阶时间翻倍

### 2.7 每日任务系统

**功能说明**：
- 任务刷新：每天自动刷新5个任务
- 任务品质：任务分为不同品质，品质越高奖励越好
- 任务完成：完成任务获得奖励
- 任务刷新：使用灵石刷新任务品质

**任务类型**：
- 炼丹任务：炼制指定数量的丹药
- 战斗任务：击败指定数量的怪物
- 体力任务：消耗指定数量的体力
- 物品任务：消耗指定数量的物品

**任务奖励**：
- 灵石：游戏主要货币
- 修为：提升境界所需
- 强化石：用于装备强化
- 洗练石：用于装备洗炼

### 2.8 封神榜系统

**功能说明**：
- 玩家排名：根据玩家修为和境界进行排名
- 排名奖励：根据排名获得不同奖励
- 实时更新：玩家修为变化时自动更新排名

**排名规则**：
- 优先按境界排序，境界高的排名靠前
- 同境界按修为排序，修为高的排名靠前
- 每个玩家在榜单中只占据一个位置
- 排名提升时，前面的玩家会自动后移

**奖励机制**：
- 每日根据排名发放奖励
- 排名越靠前，奖励越丰厚
- 包含灵石、强化石、特殊物品等

### 2.9 其他系统

**功法系统**：
- 功法学习：学习各种功法提升属性
- 功法参悟：提升功法等级，增加效果
- 功法装备：装备功法获得属性加成

**灵田系统**：
- 灵田种植：种植灵草和灵药
- 灵田升级：提升灵田等级，增加产量
- 灵草收获：收获灵草用于炼丹

**宗门系统**：
- 加入宗门：选择加入一个宗门
- 宗门任务：完成宗门任务获得贡献
- 宗门兑换：使用贡献兑换物品
- 弟子分配：分配弟子进行各种工作

**挑战塔系统**：
- 层数挑战：挑战不同层数的敌人
- 奖励获取：每层都有对应奖励
- 排行榜：记录最高挑战层数

**仙武台系统**：
- PVP对战：与其他玩家进行对战
- 排名系统：根据对战结果排名
- 奖励发放：根据排名获得奖励

**成就系统**：
- 成就解锁：完成特定任务或达到特定条件解锁成就
- 成就奖励：解锁成就后获得奖励
- 成就等级：成就分为不同等级，等级越高奖励越丰厚

**签到系统**：
- 每日签到：每天登录游戏进行签到
- 签到奖励：签到获得各种奖励
- 连续签到：连续签到天数越多，奖励越丰厚

**礼包系统**：
- 礼包领取：领取各种游戏礼包
- 礼包类型：包括新手礼包、每日礼包、活动礼包等
- 礼包内容：包含灵石、强化石、特殊物品等

**坊市系统**：
- 物品购买：使用灵石购买各种物品
- 物品出售：出售不需要的物品获得灵石
- 刷新商品：刷新坊市中的商品列表
- 稀有商品：随机出现稀有商品，需要及时购买

## 3. 游戏界面

### 3.1 主界面

**布局**：
- 顶部：用户信息、资源栏（体力、仙晶、灵石）
- 中部：角色信息、装备栏、修炼进度条、修炼控制按钮
- 底部：功能按钮（剧情、背包、功法、历练、丹炉、灵宠、灵田、炼体）
- 右侧：菜单按钮（每日任务、礼包、签到、成就、封神榜、坊市、宗门、万妖塔、仙武台、设置）

**功能按钮**：
- 开始修炼/修炼效率：控制修炼状态和查看修炼速度
- 突破：尝试突破到更高境界
- 各种功能系统入口：进入对应系统界面

**布局**：
- 左侧：装备部位选择
- 右侧：对应部位的装备列表

- 灵宠详情：显示灵宠属性、等级、技能等

**操作**：
- 点击灵宠查看详情

**操作**：

- 输入炼制数量

### 3.5 每日任务界面

**布局**：
- 任务列表：显示当前任务
- 任务详情：显示任务目标、进度、奖励

**操作**：
- 点击任务查看详情
- 点击「完成」按钮领取奖励
- 点击「刷新」按钮刷新任务品质
- 点击「添加任务」按钮增加任务数量

## 4. 游戏机制

### 4.1 资源系统

**主要资源**：
- 灵石：游戏主要货币，用于购买物品、强化装备等
- 仙晶：高级货币，用于购买稀有物品、加速等
- 体力：用于探索、挑战等活动
- 修为：提升境界所需
- 灵气：用于功法修炼
- 贡献：用于宗门兑换

**获取途径**：
- 灵石：探索、任务、出售物品
- 仙晶：签到、活动、充值
- 体力：自动恢复、使用体力丹
- 修为：修炼、使用修为丹
- 灵气：探索、使用灵气丹
- 贡献：完成宗门任务

### 4.2 战斗系统

**战斗机制**：
- 自动战斗：系统自动计算战斗结果
- 战力计算：根据角色属性、装备、灵宠等计算战力
- 胜负判定：战力高的一方获胜

**战斗类型**：
- 探索战斗：探索秘境时的战斗
- 挑战塔战斗：挑战万妖塔时的战斗
- PVP战斗：与其他玩家的对战

### 4.3 成长系统

**成长途径**：
- 境界提升：通过修炼获得修为，突破境界
- 装备强化：提升装备等级和属性
- 灵宠培养：提升灵宠等级和属性
- 功法学习：学习和提升功法
- 炼体修炼：提升肉体强度

**属性计算**：
- 基础属性：角色初始属性
- 装备加成：装备提供的属性
- 灵宠加成：出战灵宠提供的属性
- 功法加成：装备的功法提供的属性
- 境界加成：当前境界提供的属性

### 4.4 社交系统

**社交功能**：
- 封神榜：查看玩家排名
- 宗门：加入宗门，与其他玩家互动
- PVP：与其他玩家对战

**互动方式**：
- 宗门任务：与宗门成员一起完成任务
- PVP对战：挑战其他玩家
- 排行榜：查看其他玩家的进度

## 5. 游戏配置

### 5.1 配置文件

**主要配置**：
- `gameConfig.js`：游戏核心配置，包括境界、装备、丹药等
- `identities.json`：身份配置，包括不同身份的属性加成

**配置项**：
- 境界配置：各境界的名称、所需修为、属性加成
- 装备配置：装备品质、属性范围、强化规则
- 丹药配置：丹药配方、材料需求、效果
- 灵宠配置：灵宠属性、成长率、技能
- 任务配置：任务类型、奖励、刷新规则

### 5.2 存储系统

**存储方式**：
- 使用localStorage存储游戏数据

- 手动保存：页面关闭时自动保存

**存储内容**：

## 6. 开发说明

### 6.1 技术栈

**前端技术**：
- HTML5：页面结构
- CSS3：页面样式
- JavaScript：游戏逻辑
- localStorage：数据存储

**项目结构**：
- `src/`：源代码目录
  - `css/`：样式文件
  - `js/`：JavaScript文件
  - `data/`：游戏配置
- `image/`：图片资源
- `index.html`：主页面

### 6.2 系统架构

**模块化设计**：
- 每个系统独立成模块
- 通过GameEngine整合所有系统

## 7. 常见问题

### 7.1 游戏进度丢失

**原因**：
- 浏览器清除缓存
- 隐私模式关闭
- localStorage存储空间不足

**解决方法**：
- 定期手动备份游戏数据
- 避免在隐私模式下长时间游戏
- 清理浏览器缓存，释放存储空间

### 7.2 游戏卡顿

**原因**：
- 浏览器性能问题
- 页面元素过多
- 脚本执行时间过长

**解决方法**：
- 关闭其他浏览器标签页
- 刷新页面
- 使用性能更好的浏览器

### 7.3 功能异常

**原因**：
- 数据结构错误
- 脚本执行错误
- 配置文件错误

**解决方法**：
- 刷新页面
- 清除游戏数据，重新开始
- 检查浏览器控制台是否有错误信息

**修炼效率计算**：
```javascript
// 基础修炼效率 = baseExpGain
const baseEfficiency = baseExpGain;
// 获取功法加成
const skillBonus = this.engine.skillSystem ? this.engine.skillSystem.calculateTotalBonus() : { cultivationSpeed: 0 };
const skillAddition = skillBonus.cultivationSpeed || 0;
// 获取跟脚加成
const genJiaoAddition = this.state.data.genJiao ? (this.state.data.genJiao.efficiency - 1) : 0;
// 当前修炼效率 = 基础修炼效率 * (跟脚加成 + 功法加成)
const totalBonus = 1 + genJiaoAddition + skillAddition;
const expGain = Math.round(baseEfficiency * totalBonus);
```

```javascript
const baseRate = 0.1; // 基础成功率10%
const furnaceBonus = this.state.data.alchemy.furnace.bonus;

// 使用功法系统的calculateTotalBonus获取炼丹成功率加成

if (this.engine && this.engine.skillSystem) {
```
